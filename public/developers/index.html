Introducción

La MetroProp Partner API v1 permite a CRMs y sistemas de gestión inmobiliaria integrar sus datos con la plataforma MetroProp. A través de esta API podrás:

  • Crear organizaciones inmobiliarias con su sucursal y administrador
  • Publicar, actualizar y eliminar propiedades
  • Gestionar imágenes y archivos adjuntos
  • Desactivar propiedades sin eliminarlas

Base URL:

/api/partner/v1

Autenticación

Todos los endpoints requieren autenticación via API Key + API Secret enviados en los headers HTTP. Las credenciales se proporcionan al registrar un partner.

Headers requeridos

HeaderDescripciónEjemplo
x-api-keyAPI Key del partnerpk_live_abc123def456
x-api-secretAPI Secret del partnersk_live_xyz789ghi012
Importante: Nunca expongas tu API Secret en código frontend o repositorios públicos. Todas las llamadas deben realizarse desde tu backend.

Rate Limiting

La API tiene un límite de 100 requests por minuto por partner. Si excedes el límite recibirás un error 429 Too Many Requests.

API Playground (Swagger)

Utilizá el playground interactivo para probar los endpoints directamente desde el navegador. Ingresá tu API Key y Secret en el botón "Authorize" y podrás hacer POST, GET, PUT y DELETE reales.

▶ Abrir API Playground (Swagger UI)
Tip: En Swagger UI, hacé click en el botón "Authorize" (candado) arriba a la derecha e ingresá tu x-api-key y x-api-secret para que todos los requests se autentiquen automáticamente.

POST Crear Organización

POST /api/partner/v1/organizations

Crea una organización inmobiliaria con su sucursal predeterminada y un usuario administrador. El admin recibe un email de bienvenida para verificar su cuenta.

Respuesta: organization_id, branch_id y admin_user_id — usá branch_id como branch_reference_id al crear propiedades.

POST Crear Propiedad

POST /api/partner/v1/properties

Crea una nueva propiedad vinculada a un branch_reference_id. Si ya existe una propiedad con el mismo reference_code en la organización, se actualiza automáticamente (upsert). Incluye soporte para videos y multimedia 360.

Imágenes y adjuntos se gestionan mediante endpoints dedicados: POST /uploadImage y POST /uploadAttached (multipart/form-data).

PUT Actualizar Propiedad

PUT /api/partner/v1/properties/{referenceCode}

Actualiza campos escalares, operaciones y tags. Solo enviar los campos que se desean modificar.

GET Obtener Propiedad

GET /api/partner/v1/properties/{referenceCode}

Retorna todos los datos de la propiedad, incluyendo multimedia, operaciones y tags.

DELETE Eliminar Propiedad

DELETE /api/partner/v1/properties/{referenceCode}

Soft delete: marca la propiedad como eliminada sin borrar datos físicamente.

POST Desactivar Propiedad

POST /api/partner/v1/properties/{referenceCode}/deactivate

Cambia el estado a NO_DISPONIBLE (4). La propiedad queda visible pero inactiva.

POST Subir Imagen

POST /api/partner/v1/properties/{referenceCode}/uploadImage

Sube un archivo de imagen directamente a la propiedad via multipart/form-data. Retorna inmediatamente con un image_reference_id y upload_status=pending. El archivo se procesa y sube a S3 en background.

Campos del form

CampoTipoRequeridoDescripción
filebinarySíArchivo de imagen (jpg, png, webp, etc.)
descriptionstringNoDescripción de la imagen
order_positionintegerNoPosición de orden (0+)
is_blueprintbooleanNoSi es un plano (true / false)

PATCH Actualizar Metadata Imagen

PATCH /api/partner/v1/properties/{referenceCode}/image/{imageReferenceCode}

Actualiza descripción, posición o flag de plano de una imagen existente. No reemplaza el archivo. Usar el image_reference_id retornado al subir.

DELETE Eliminar Imagen

DELETE /api/partner/v1/properties/{referenceCode}/image/{imageReferenceCode}

Elimina una imagen usando su image_reference_id (ID numérico retornado al subir la imagen).

POST Subir Adjunto

POST /api/partner/v1/properties/{referenceCode}/uploadAttached

Sube un archivo adjunto directamente a la propiedad via multipart/form-data. Retorna inmediatamente con un attached_reference_id y upload_status=pending. El archivo se procesa y sube a S3 en background.

Campos del form

CampoTipoRequeridoDescripción
filebinarySíArchivo adjunto (pdf, doc, xls, etc.)
descriptionstringNoDescripción del adjunto
orderintegerNoPosición de orden (0+)

PATCH Actualizar Metadata Adjunto

PATCH /api/partner/v1/properties/{referenceCode}/attached/{attachedReferenceCode}

Actualiza descripción u orden de un adjunto. No reemplaza el archivo. Usar el attached_reference_id retornado al subir.

DELETE Eliminar Adjunto

DELETE /api/partner/v1/properties/{referenceCode}/attached/{attachedReferenceCode}

Elimina un adjunto usando su attached_reference_id (ID numérico retornado al subir el adjunto).

Ejemplo: Subir Imagen

POST /api/partner/v1/properties/PROP-DEMO-001/uploadImage

Enviar como multipart/form-data. Ejemplo con curl:

# curl curl -X POST \ -H 'x-api-key: tu_api_key' \ -H 'x-api-secret: tu_api_secret' \ -F 'file=@/ruta/a/imagen.jpg' \ -F 'description=Living comedor' \ -F 'order_position=1' \ -F 'is_blueprint=false' \ 'https://api.metroprop.com/api/partner/v1/properties/PROP-DEMO-001/uploadImage'
// Response (201 Created) { "success": true, "data": { "image_reference_id": 42, // ← usar para PATCH y DELETE "upload_status": "pending", "informacion_adicional": "Recordá utilizar este ID de referencia para eliminar o modificar esta imagen en el portal" } }
El upload_status cambia a completed cuando S3 termina el procesamiento. Si hay error, cambia a failed y se incluye error_message. Podés consultar el estado con PATCH (que retorna el status actual).
