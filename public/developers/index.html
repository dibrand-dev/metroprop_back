Introducción

La MetroProp Partner API v1 permite a CRMs y sistemas de gestión inmobiliaria integrar sus datos con la plataforma MetroProp. A través de esta API podrás:

  • Crear organizaciones inmobiliarias con su sucursal y administrador
  • Publicar, actualizar y eliminar propiedades
  • Desactivar propiedades sin eliminarlas

Base URL:

/api/partner/v1

Autenticación

Todos los endpoints requieren autenticación via API Key + API Secret enviados en los headers HTTP. Las credenciales se proporcionan al registrar un partner.

Headers requeridos

HeaderDescripciónEjemplo
x-api-keyAPI Key del partnerpk_live_abc123def456
x-api-secretAPI Secret del partnersk_live_xyz789ghi012
Importante: Nunca expongas tu API Secret en código frontend o repositorios públicos. Todas las llamadas deben realizarse desde tu backend.

Rate Limiting

La API tiene un límite de 100 requests por minuto por partner. Si excedes el límite recibirás un error 429 Too Many Requests.

API Playground (Swagger)

Utilizá el playground interactivo para probar los endpoints directamente desde el navegador. Ingresá tu API Key y Secret en el botón "Authorize" y podrás hacer POST, GET, PUT y DELETE reales.

▶ Abrir API Playground (Swagger UI)
Tip: En Swagger UI, hacé click en el botón "Authorize" (candado) arriba a la derecha e ingresá tu x-api-key y x-api-secret para que todos los requests se autentiquen automáticamente.

POST Crear Organización

POST /api/partner/v1/organizations

Crea una organización inmobiliaria con su sucursal predeterminada y un usuario administrador. El admin recibe un email de bienvenida para verificar su cuenta.

▶ Respuesta importante: El endpoint retorna un branch_id. Conservá este valor y usalo como branch_reference_id al crear propiedades, para asignarlas a la sucursal deseada de esa organización.

POST Retry Uploads

POST /api/partner/v1/properties/{referenceCode}/retry-uploads

Reintenta subir a S3 todos los archivos (imágenes y adjuntos) que fallaron previamente para una propiedad específica del partner. Útil si hubo errores de red o problemas temporales con S3.

  • Parámetro: referenceCode — Código de referencia de la propiedad
Nota: Solo reintenta uploads fallidos. Si todos los archivos ya están subidos correctamente, no realiza cambios.
// Request POST /api/partner/v1/properties/PROP-DEMO-001/retry-uploads Headers: x-api-key, x-api-secret
// Response (202 Accepted) { "success": true, "message": "Reintentos en progreso para archivos multimedia fallidos." }

PUT Actualizar Propiedad

PUT /api/partner/v1/properties/{referenceCode}

Actualiza campos escalares, operaciones y tags. Solo enviar los campos que se desean modificar.

⚠ Importante: Si se incluye alguno de los campos images, attached, tags, videos o multimedia360, reemplazará completamente el set existente en la propiedad. Si no hay cambios, simplemente no incluyas el campo en el request. Si deseas eliminar todos los elementos de alguno de esos campos, envía un array vacío (ej: "tags": [] para eliminar todas las tags asociadas).

Ejemplo

// Request Body — solo los campos que se quieren modificar { // Actualizar precio y estado "price": 125000, "status": 3, // Actualizar descripción "description": "Departamento remodelado con cocina americana y balcón amplio. Excelente ubicación.", // Tags: En caso de enviar tags, incluir TODOS los que debe tener la propiedad "tags": [1, 3, 7] }
// Response (200 OK) { "success": true, "data": { "id": 108, "reference_code": "PROP-DEMO-001", "price": 125000, "status": 3 } }

GET Obtener Propiedad

GET /api/partner/v1/properties/{referenceCode}

Retorna todos los datos de la propiedad, incluyendo multimedia, operaciones y tags.

DELETE Eliminar Propiedad

DELETE /api/partner/v1/properties/{referenceCode}

Soft delete: marca la propiedad como eliminada sin borrar datos físicamente.

POST Desactivar Propiedad

POST /api/partner/v1/properties/{referenceCode}/deactivate

Cambia el estado a NO_DISPONIBLE (4). La propiedad queda visible pero inactiva.

POST Crear Emprendimiento

POST /api/partner/v1/developments

Crea un nuevo emprendimiento vinculado a un branch_reference_id. Si ya existe un emprendimiento con el mismo reference_code para esa organización, se actualiza automáticamente (upsert). Soporta los mismos campos de multimedia que propiedades más campos exclusivos del proyecto.

Diferencia con propiedades: Los emprendimientos representan proyectos en desarrollo (torres, barrios cerrados, complejos de oficinas). Se identifican internamente con is_development: true y tienen campos exclusivos como development_type, fecha de entrega y conteo de unidades.

GET Obtener Emprendimiento

GET /api/partner/v1/developments/{referenceCode}

Retorna todos los datos del emprendimiento incluyendo multimedia, tags y campos exclusivos de desarrollo.

  • Parámetro: referenceCode — Código de referencia del emprendimiento

PUT Actualizar Emprendimiento

PUT /api/partner/v1/developments/{referenceCode}

Actualiza campos del emprendimiento. Solo enviar los campos que se desean modificar.

⚠ Importante: Si se incluye alguno de los campos images, attached, tags, videos o multimedia360, reemplazará completamente el set existente. Enviá un array vacío (ej: "tags": []) para eliminar todos los elementos de ese campo.

DELETE Eliminar Emprendimiento

DELETE /api/partner/v1/developments/{referenceCode}

Soft delete: marca el emprendimiento como eliminado sin borrar datos físicamente. La operación no es reversible desde la API.

  • Parámetro: referenceCode — Código de referencia del emprendimiento

POST Crear / Actualizar Unidad

POST /api/partner/v1/developments/{referenceCode}/units

Crea una unidad (propiedad hija) vinculada al emprendimiento identificado por referenceCode. Si ya existe una unidad con el mismo reference_code bajo ese emprendimiento, se actualiza (upsert). El servidor asigna automáticamente development_id, organization_id y branch_id del emprendimiento padre. La propiedad una vez creada puede ser modificada utilizando el endpoint de propiedades regular PUT /properties/{referenceCode} o el endpoint de unidades PUT /developments/{referenceCode}/units/{unitReferenceCode}.

Concepto: una unidad es una propiedad normal (is_development = false) que tiene development_id apuntando al emprendimiento padre. Comparte la misma tabla properties.

GET Listar Unidades

GET /api/partner/v1/developments/{referenceCode}/units

Retorna todas las unidades activas del emprendimiento, incluyendo multimedia y tags de cada una.

// Response (200 OK) { "success": true, "total": 3, "data": [ /* array de unidades */ ] }

PUT Actualizar Unidad

PUT /api/partner/v1/developments/{referenceCode}/units/{unitRefCode}

Actualiza campos de una unidad. Solo enviar los campos que se desean modificar.

  • referenceCode: Código de referencia del emprendimiento padre
  • unitRefCode: Código de referencia de la unidad

DELETE Eliminar Unidad

DELETE /api/partner/v1/developments/{referenceCode}/units/{unitRefCode}

Soft delete: marca la unidad como eliminada sin borrar datos físicamente.

  • referenceCode: Código de referencia del emprendimiento padre
  • unitRefCode: Código de referencia de la unidad

GET Países

GET /api/partner/v1/locations/countries

Retorna la lista completa de países disponibles en la plataforma. Usar el id del resultado como country_id en organizaciones y propiedades.

[ { "id": 1, "name": "Argentina" }, { "id": 2, "name": "Uruguay" }, ... ]

GET Provincias / Estados

GET /api/partner/v1/locations/states?countryId={countryId}

Retorna las provincias o estados de un país dado su country_id. Usar el id del resultado como state_id.

ParámetroTipoDescripción
countryIdinteger (query)ID del país (obtenido en el endpoint anterior)

GET Localidades / Ciudades

GET /api/partner/v1/locations/cities?stateId={stateId}

Retorna las localidades o ciudades de una provincia dado su state_id. Usar el id del resultado como location_id.

ParámetroTipoDescripción
stateIdinteger (query)ID de la provincia/estado

GET Sublocalidades / Barrios

GET /api/partner/v1/locations/sub-locations?locationId={locationId}

Retorna barrios o sublocalidades de una localidad dado su locationId. Usar el id del resultado como sub_location_id / sublocation_id.

// Request Body — todos los campos posibles
